# Handoff — Morning Ben

## Single Grasp

Morning Ben — телеграм-бот с утренними историями вместо рилсов: жмёшь
«Дальше ▶», получаешь короткие факты-сообщения. **MVP (стадии А+Б) сделан
целиком и уже работает на сервере firstvds-amsterdam в tmux** — бот
отвечает, вся генерация на вотчере: он держит очередь непоказанных не ниже
15, догенерируя пачками по 5 (столько, сколько реально читается). Сверху
добавлена обратная связь: любой свободный текст боту (не кнопка и не
команда) копится в `state/feedback.json`, три самых свежих сообщения с
датами уходят в промпт генератора. **Плюс картинки из Pinterest**
(mcp-pinterest в `vendor/`): зрелищные темы — до 5 «вайбовых» картинок по
английским запросам («norway aesthetic»), научпоп — до 2 схем; одна
картинка = одно нажатие «Дальше», между текстами. Всё задеплоено и
проверено вживую end-to-end. Следующий шаг: почитать бота пару утр —
теперь с картинками — и написать ему пожелания в чат.

## Context

Личный проект Ольги (единственный пользователь и стейкхолдер). Утром в
период «тупняка» рука тянется к рилсам; замена — короткие персонализированные
научпоп-истории в телеграме, читаются в том же режиме листания, но оставляют
что-то полезное. Дальше по бэклогу (R2+): реальные источники и картинки,
голос Бена Гросса, память реакций, синк с Obsidian. Спека:
`docs/superpowers/specs/2026-07-19-morning-ben-mvp-design.md`.

## Goal

MVP = компонент А (бот-доставка) + Б-simple (генерация из знаний модели).
Всё реализовано и задеплоено. Ключевые требования, которые соблюдены:
- контент генерирует **только sonnet** по промптам из папки + истории
  показанного (антиповтор), ничего не мокается;
- истории копятся, старые не стираются; свежие показываются первыми;
- бот отвечает только владелице (OWNER_CHAT_ID из .env).

## Working files

| Файл | Что это |
|------|---------|

## Progress narrative

За одну сессию 2026-07-19: написан и оттестирован бот (все сценарии спеки
проверены реальными сообщениями через telegram MCP от имени Ольги:
пустая очередь, полное дочитывание с `completed:true`, скип с
`completed:false`, «закончились» ровно в конце, подхват нового файла без
рестарта, восстановление позиции после убийства процесса). Написан
generate.sh, первые истории сгенерены соннетом. Затем по фидбеку: N как
параметр; генератор передаёт в промпт ещё и темы непоказанных историй
(дыра в антиповторе спеки); воркфлоу-обёртка; правка промпта «изобретай
темы около направлений, не бери примеры дословно» (сразу дало эффект:
«Дания: дети спят в колясках у кафе»). Деплой: сначала systemd user-units,
потом по просьбе Ольги переехали на tmux (start_all.sh). В конце сессии
убрали ночной прогон на 20 историй (nightly_generate.sh, был по её же
просьбе «в 21:00 мск 20 штук») — Ольга выбрала единый механизм: только
вотчер с порогом 15. Вотчер проверен вживую на сервере.

Вторым заходом того же дня добавлена обратная связь: в `bot.py` — хендлер
на любой свободный текст (регистрируется после кнопок, `~filters.COMMAND`),
пишет `{"date", "text"}` в `state/feedback.json` и отвечает «Принял 📝»;
в `generate.sh` и воркфлоу — раздел «Обратная связь от Ольги» в промпте:
только три самых свежих записи, свежие первыми, с датами. Проверено
end-to-end через telegram MCP (сообщение → ответ бота → запись на сервере →
выборка в промпт); тестовая запись после проверки стёрта. По ходу деплоя
уронил всю tmux-сессию через `pkill -f bot.py` (см. What didn't work),
поднял обратно `./start_all.sh`.

Третьим заходом того же дня — картинки (R2 частично). Сабагент оттестировал
`terryso/mcp-pinterest` в песочнице: headless Chrome скрапит выдачу
Pinterest, ключей не нужно, поиск ~11-25с, отдаёт прямые живые ссылки
`i.pinimg.com`, хотлинк не блокируется. По результатам: `setup_pinterest.sh`
(вендоринг + патчи), правила картинок в `generate_stories.md` (зрелищное —
до 5 aesthetic по-английски, научпоп — до 2 схем, не подряд, не на краях,
`/originals/`→`/736x/`), поддержка image-элементов в `bot.py`. На сервере
Chrome не заводился (нет libnss3 и её компании, sudo нет) — решено
распаковкой deb-пакетов в `~/chrome-libs` + `LD_LIBRARY_PATH`. Проверено
end-to-end: соннет на сервере сам сходил в Pinterest, собрал истории с
картинками по правилам (и корректно оставил научпоп-историю без картинок,
когда схем не нашлось), бот прислал фото в телеграм между текстами.
Тестовые сообщения стёрты, state восстановлен из бэкапа.

## What worked

- **Тестирование через mcp-telegram от имени юзера** — единственный способ
  проверить бота end-to-end без участия Ольги; get_messages показывает
  bold-entity обратно как `**…**` — это признак успеха, не сырые звёздочки
  (проверять `grep 'HTML-разметка не прошла' logs/bot.log`).
- **Очередь «на лету» из файловой системы** (спека) — все сбои вырождаются
  в «показываем меньше», ничего не падает; подхват новых файлов бесплатный.
- **`{{N_STORIES}}`-плейсхолдер + sed** — параметризация промпта без
  усложнения.
- **Темы непоказанных историй в промпт генератора** — иначе свежие прогоны
  дублируют то, что уже лежит в очереди (shown_history пуст по утрам).
- **Вотчер с порогом как единственный генератор** — очередь сама держится
  на уровне, генерится ровно столько, сколько читается; расписания не нужны.
- **Pending-апдейты телеграма переживают даунтайм бота** — сообщение,
  отправленное пока бот лежал, обработалось сразу после старта; можно
  спокойно перезапускать, ничего не теряется.
- **Chrome без sudo**: недостающие библиотеки — `apt-get download` (работает
  без sudo) + `dpkg -x` в `~/chrome-libs` + `LD_LIBRARY_PATH`; сам Chrome —
  `npx @puppeteer/browsers install chrome@stable` в `~/.cache/puppeteer`.
  Всё уже автоматизировано в `setup_pinterest.sh`.
- **Хотлинк `i.pinimg.com` в reply_photo** — телеграм сам забирает картинку
  по URL, скачивать на диск не нужно; `/736x/`-вариант живой всегда
  (в отличие от `/originals/`, который в ~5-20% отдаёт 403).
- **Первый Pinterest-поиск после старта сервера бывает «холостым»** (0
  результатов без ошибки) — в промпте генератора заложен один повтор.

## What didn't work

- **Воркфлоу по имени в той же сессии**: `Workflow({name})` не видит только
  что созданный файл — реестр собирается на старте сессии. Запускать по
  `scriptPath`.
- **systemd** — работал нормально, но Ольга попросила без него («сервер не
  перезагружается»): теперь tmux; не возвращаться к systemd без запроса.
- **Cron/OS-планировщики Ольге не нравятся** — регулярность делать внутри
  скриптов (вечные циклы) или средствами Claude.
- **CronCreate в Claude-сессии — не персистентен** (умирает с сессией,
  максимум 7 дней) — годится только как времянка.
- **Хайку-агент на валидацию JSON** — убрали как лишнюю трату; генератор
  сам проверяет `python3 -m json.tool`, плюс бот валидирует при чтении.
- **`pkill -f bot.py` на сервере** — убивает и tmux-сервер (в его argv
  есть строка запуска окна с «bot.py»), падает вся сессия вместе с
  вотчером. Перезапускать только штатно:
  `tmux kill-session -t morning-ben && ./start_all.sh`.
- **mcp-pinterest из коробки на VPS** — путь к Chrome захардкожен
  (`/usr/bin/google-chrome`), env-переопределения в апстриме нет; плюс
  сервер гадит не-JSON строками в stdout. Оба места патчит
  `setup_pinterest.sh`; при обновлении вендора (сейчас пин b2b1b39)
  патчи надо перепроверить.
- **Сортировка очереди по `generated_at` — лексикографическая**, а соннет
  пишет даты то как `…Z`, то как `…+02:00`: свежая история может встать
  в очереди не первой. На поведение почти не влияет (очередь всё равно
  вычитывается), но при отладке «почему бот показал не то» помнить.

  `logs/bot.log`, `logs/watcher.log`, `logs/generate.log`; перезапуск —
  `tmux kill-session -t morning-ben && ./start_all.sh`.
